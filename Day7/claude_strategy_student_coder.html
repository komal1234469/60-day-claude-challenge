Biggest mistakes to avoid
Using Opus for every task — it's slow and overkill for daily coding. Save it for architecture or interview prep only.
Pasting code without context — always explain what the code is supposed to do and what error you're seeing.
Accepting code without understanding it — ask Claude to explain every line. You're a student; learning beats copy-pasting.
Using Max effort for trivial tasks — wastes time and context. Match effort to task complexity.
One-shot giant prompts — break big problems into steps. "Build me a full app" gets worse results than iterative building.
Final recommendation
If you could only pick one: Claude Sonnet 4.6 at Standard effort.
It handles 90% of student coding tasks — debugging, learning, building features — with the best balance of speed, depth, and reliability. Switch to High effort when stuck on hard problems, and pull out Opus only for your most important work like capstone projects or placement interviews.
